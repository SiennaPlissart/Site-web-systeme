  1. Définition de la virtualisation : La virtualisation permet de diviser les ressources matérielles d'un seul ordinateur (processeurs, mémoire, stockage et autres) en plusieurs ordinateurs virtuels, appelés machines virtuelles (VM).
  2. Différence entre émulateur, simulateur et outil de virtualisation :
    • Émulateur : Reproduit le fonctionnement d'un système matériel ou logiciel différent, souvent utilisé pour tester des systèmes ou applications sur une architecture différente (par exemple, émuler un téléphone Android sur un PC).
    • Simulateur : Imite le comportement d'un système pour tester des concepts ou des processus sans reproduire complètement le matériel sous-jacent.
    • Outil de virtualisation : Permet l'exécution de plusieurs environnements d'exploitation sur une même machine physique en utilisant un hyperviseur pour partager les ressources matérielles (ex. : VirtualBox, VMware Workstation).
  3. Type de virtualisation pour les environnements VirtualBox, VMware Workstation et Stratus :
    • VirtualBox : Virtualisation de type 2 (basée sur un hyperviseur installé sur un système d'exploitation hôte).
    • VMware Workstation : Virtualisation de type 2, similaire à VirtualBox, utilisant un hyperviseur hébergé.
    • Stratus : Virtualisation de type 1 (basée sur un hyperviseur directement installé sur le matériel, si on parle de solutions Stratus hautement disponibles comme Stratus ftServer).
  4. 4e Question : Où avez-vous choisi d’enregistrer la machine virtuelle ?

    Nous avons choisi d’enregistrer la machine virtuelle localement sur notre machine. Cette décision a été prise pour assurer une meilleure accessibilité et éviter toute dépendance à une connexion réseau externe. L'enregistrement local garantit également une gestion plus rapide des ressources et un contrôle total sur les fichiers de la machine virtuelle.
